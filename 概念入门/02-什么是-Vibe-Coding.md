# 什么是 Vibe Coding？

Vibe Coding（氛围编程）= **描述你的需求，AI 来写代码**。

你只需要「说人话」，代码的事交给 AI。

## 类比：Vibe Coding vs 传统 Coding

| 对比 | 传统 Coding | Vibe Coding |
|------|-------------|-------------|
| **写代码** | 程序员手敲每一行 | AI 根据描述生成代码 |
| **需求表达** | 写详细的需求文档 | 说一句话或描述场景 |
| **调试** | 人工 printf 调试 | AI 分析报错、直接修复 |
| **学习门槛** | 需要懂编程语言 | 会说话就会编程 |

## 为什么这是未来？

1. **降门槛**：不需要学编程，也能做数字产品
2. **提效率**：10 倍速开发，从想法到原型
3. **专注创意**：你负责「想做什么」，AI 负责「怎么实现」
4. **民主化**：让更多人能参与到软件开发中

## 如何做 Vibe Coding（4步）

### 第 1 步：清晰描述需求

❌ 模糊：「帮我做个网站」
✅ 清晰：「帮我做一个个人博客，有首页、关于页和文章列表页，使用深色主题」

### 第 2 步：让 AI 先出方案

不要直接让 AI 写代码，先让它给你设计方案：
> 「帮我设计一个中小型电商网站的技术架构」

### 第 3 步：分步骤实现

不要一次让 AI 写整个系统，分模块来：
- 先做用户登录注册
- 再做商品展示
- 最后做购物车和支付

### 第 4 步：审查和调整

AI 写的代码不一定完美，你需要：
- 理解代码在做什么
- 提出修改意见
- 让 AI 调整优化

## 适合场景

- 快速原型开发
- 个人项目、小工具
- 学习新技术时的 Demo
- 重复性代码生成

## 不适合场景

- 复杂系统架构（仍需要专业程序员）
- 性能敏感的底层代码
- 安全性要求极高的场景

---

**一句话理解**：Vibe Coding 就是「说你所想，AI 编所写」，把编程从技术活变成表达活。

# 什么是 GitHub？

GitHub 是**代码的云端仓库 + 社交平台 + 版本控制工具**。

> 注意：读作「GitHub」，不是「gayhub」！就像Git不是「给特」一样。

## 类比：云盘 + 朋友圈 + 版本控制

### 云盘
GitHub 就像一个特殊的云盘，专门存放代码文件。你可以上传、下载、同步代码。

### 朋友圈
你可以「关注」其他开发者，看他们在做什么项目。可以给他们点赞、评论、参与讨论。

### 版本控制
这是 GitHub 最核心的能力——**记录代码的每一次修改**。

就像 Word 的「修订模式」：
- 每次修改都有记录
- 可以回滚到任意版本
- 多人同时修改不会覆盖

## 核心概念

### Repository（仓库）
代码的「文件夹」，一个项目对应一个仓库。
- `repo/` = 存放代码的地方
- 可以设置公开（Public）或私有（Private）

### Commit（提交）
代码的「存档点」。每次提交都会记录：
- 这次改了哪些内容
- 改了什么理由（commit message）
- 谁改的

### Branch（分支）
从主线分出来的「平行宇宙」。
- 默认分支：`main` 或 `master`
- 可以开新分支做实验，不影响主线
- 实验成功后再合并回主线

### Pull Request（PR）
请求把分支的修改「拉」到主线。
- 发 PR = 申请合并代码
- 其他人可以 review（审查）
- 讨论没问题后才能合并

## 为什么我们需要它？

### 1. 代码备份
代码存在云端，电脑坏了也不丢。

### 2. 团队协作
多人同时开发，分支管理，合并代码，不再「你覆盖了我的文件」。

### 3. 代码复用
开源项目可以直接「Fork」过来用，站在巨人的肩膀上。

### 4. 简历加成
GitHub 主页 = 程序员的履历，展示你的代码能力和项目经验。

## 常见操作速查

| 操作 | 命令 | 说明 |
|------|------|------|
| 克隆到本地 | `git clone` | 把仓库下载到电脑 |
| 拉取更新 | `git pull` | 把远程最新代码拉下来 |
| 推送代码 | `git push` | 把本地代码推送到远程 |
| 创建分支 | `git branch` | 新建一个分支 |
| 合并分支 | `git merge` | 把分支合并到当前分支 |

---

**一句话理解**：GitHub = 代码的「百度网盘 + 微博 + 历史记录」，让程序员协作更高效。

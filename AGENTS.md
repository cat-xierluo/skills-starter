# AGENTS.md - 项目规范

本项目是 Skill 创作模板仓库，为创建、测试和发布 Skill 提供标准结构。

---

## 核心原则

- **中文优先**：回复使用中文，文档使用中文。
- **文档即上下文**：关键信息写在文件里，不靠记忆。
- **先搜索，后创造**：遇到问题先查现有文档，再创新方案。

---

## 目录命名规范

- **目录名**：用中文（如 `概念入门`、`工具指南`、`创建Skill`）
- **文件名**：用中文（如 `01-什么是-Skill.md`、`02-目录结构.md`）
- **代码文件除外**：`SKILL.md`、`AGENTS.md`、`CLAUDE.md` 等保持原样

---

## 文档编写规范

### SKILL.md

Skill 定义文件，放在技能根目录。

```markdown
---
name: skill-name        # 英文，唯一标识
description: 英文描述   # 英文，说明用途
---

# Skill Name

（正文用中文编写）
```

### 其他文档

- 语言：中文
- 代码块：使用 \`\`\`
- 列表：用 `-` 或 `1.` 均可

---

## Git 工作流

### 分支命名

```
refactor/xxx   # 重构
feature/xxx   # 新功能
bugfix/xxx     # 修复
```

### Commit 规范

- 语言：中文
- 风格：简洁，说明做了什么

```
feat: 新增 xxx 功能
fix: 修复 xxx 问题
refactor: 重构 xxx
docs: 更新 xxx 文档
```

### PR 流程

1. 创建 PR
2. 常规功能：创建后自动合并
3. 重大变更：人工 review 后合并

## 其他

- 安全与合规：避免编造事实，不确定的信息标注"待确认"
- 敏感信息：API Key、Token 等不可提交到 Git
- 破坏性操作：谨慎执行，保持用户未提交的更改

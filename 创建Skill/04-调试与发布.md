# 04 调试与发布

## 本地调试方法

### 1. 直接运行脚本
```bash
# 单独跑脚本，看输出
cd <skill-name>/scripts
python3 main.py --arg value
```

### 2. 集成测试
在 OpenClaw 中触发 Skill，检查：
- 触发条件是否正确
- 参数解析是否正确
- 输出是否符合预期
- 错误处理是否完善

### 3. 常见问题

| 问题 | 可能原因 | 解决 |
|------|----------|------|
| 触发不了 | 触发关键词没配对 | 检查 SKILL.md 的 triggers |
| 权限错误 | 缺少 OAuth 授权 | 重新授权 |
| 超时 | 任务太重 | 拆解或加 timeout |
| 路径错误 | 用了相对路径 | 用 `__file__` 获取绝对路径 |

## 发布到 GitHub

### 1. 初始化（如果从模板复制）
```bash
cd <skill-name>
git init
git add .
git commit -m "feat: initial commit"
```

### 2. 创建仓库
```bash
# 在 GitHub 上创建仓库，然后
git remote add origin https://github.com/<user>/<skill-name>.git
git push -u origin main
```

### 3. 推荐仓库结构
```
<skill-name>/
├── SKILL.md           # 必须：Skill 说明
├── scripts/           # 必须：核心代码
│   ├── main.py        # 入口文件
│   └── .env.example   # 环境变量示例
├── README.md          # 可选：详细使用说明
└── LICENSE            # 推荐：MIT
```

## 如何让其他人使用

### 方式一：ClawHub（推荐）
上传到 ClawHub，让用户一键安装。

### 方式二：GitHub 链接
直接分享 GitHub 仓库地址，用户手动安装。

### 方式三：OpenClaw 导入
如果平台支持，导入 Skill 目录或仓库 URL。

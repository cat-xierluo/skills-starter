# 02 搜索现成方案

不要重复造轮子。动手之前先搜一圈。

## 为什么先搜索？

- 很可能已经有人解决了类似问题
- 站在前人肩膀上，省下的是你自己的时间
- 搜索过程中会加深对问题的理解
- 有时候会发现更优雅的解决思路

## 搜索顺序

**1. find-skill（首选）**
社区现成的 Skill，质量有保证，经过验证。

**2. GitHub**
关键词 + `openclaw skill` 或 `claude skill` 搜索。重点看 stars 和最近更新时间。

**3. 互联网**
通用问题（API 用法、概念理解）用搜索引擎。

## find-skill 使用方法

```bash
# 搜索 Skill
find-skill <关键词>

# 示例
find-skill github
find-skill "web scraping"
```

## 如何评估搜索结果

| 维度 | 看什么 | 差 | 好 |
|------|--------|-----|-----|
| 相关性 | 描述和你的需求是否匹配 | 勉强相关 | 精准命中 |
| 活跃度 | 最近更新时间 | 1年前 | 3个月内 |
| 质量 | README 是否清晰，代码是否规范 | 混乱 | 结构清晰 |
| 依赖 | 需要什么外部依赖 | 复杂 | 零依赖或轻量 |
| 许可证 | 能否商用 | 不明确 | MIT/Apache |

**找到现成方案后**：Fork → 试跑 → 评估是否满足需求 → 决定直接用还是改造。

**找不到现成方案**：恭喜，你是第一个。参考 03 基于模板创建 开始搭建。

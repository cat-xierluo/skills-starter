# 03 基于模板创建

没有现成方案时，从模板开始。

## 使用 skill-template

```bash
# 在 starter 仓库内创建一个新 skill
cp -r skills/skill-template skills/<skill-name>

# 如果要单独建仓库
cp -r skills/skill-template ../<skill-name>
```

模板目录：`skills/skill-template/`

## SKILL.md 结构解析

```markdown
---
name: your-skill-name
description: |
  本技能应在...时使用。
  不要用于：...。
license: MIT  # 示例值，发布前按实际选择
---

# Skill 名称

[简短描述：这个 Skill 做什么]

## 适用场景
[明确正向触发和负向触发]

## 快速开始
[最小可运行步骤]

## 与其他技能配合
[说明协作关系，而不是直接写死其他技能内部路径]

## 示例
[使用示例]
```

除了 `SKILL.md`，建议一起维护：

- `LICENSE.txt`
- `CHANGELOG.md`
- `ROADMAP.md`
- `TASKS.md`
- `DECISIONS.md`
- `.env.example`
- `assets/config.yaml.example`

## 如何命名 Skill

- **命名格式**：`行为-对象` 或 `对象-行为`
  - ✅ `github-issues`（行为-对象）
  - ✅ `image-understand`（对象-行为）
  - ❌ `do-something-useful`
- **用英文**（社区通用语言）
- **简短**：1-2 个单词最佳
- **清晰**：不看描述也能猜到用途

## 创建步骤

1. **复制模板**：`cp -r skills/skill-template ../<skill-name>`
2. **命名**：目录名即 Skill 名
3. **写 `SKILL.md`**：先把 `name`、`description` 和适用场景写准
4. **确认许可证**：选择许可证，补 `LICENSE.txt`，必要时在 frontmatter 写 `license`
5. **补协作文档**：确认 `CHANGELOG.md`、`ROADMAP.md`、`TASKS.md`、`DECISIONS.md` 适合当前项目
6. **实现代码**：在 `scripts/` 下写核心逻辑
7. **本地测试**：跑通基本流程
8. **调试优化**：见 04 调试与发布

# 02 搜索现成方案

不要重复造轮子。动手之前先搜一圈。

## 为什么先搜索？

- 很可能已经有人解决了类似问题
- 站在前人肩膀上，省下的是你自己的时间
- 搜索过程中会加深对问题的理解
- 有时候会发现更优雅的解决思路

## 搜索顺序

**1. 项目内置 `find-skill` 或 Skills CLI（首选）**
先用现成生态做第一轮筛选。

**2. GitHub**
关键词 + `openclaw skill` 或 `claude skill` 搜索。重点看 stars 和最近更新时间。

**3. 互联网**
通用问题（API 用法、概念理解）用搜索引擎。

## `find-skills` 的三种用法

### 方式一：直接用项目内置 `find-skill`

如果你正在这个 starter 仓库里使用 Claude Code，本项目已经按 `legal-skills` 的方式把根目录 `skills/` 暴露给 `.claude/skills`：

```text
.claude/skills -> ../skills
.claude/skills/find-skill/
```

也就是说，这个仓库本身就带了一个开箱即用的搜索引导 Skill，而实际维护的源文件是 `skills/find-skill/SKILL.md`。

### 方式二：直接用 Skills CLI 搜索

```bash
npx skills find <关键词>
```

### 方式三：先安装独立 `find-skills` 项目

如果你的工作流依赖独立 Skill 版本，先按你选用仓库的 README 安装：

```bash
npx skills add <仓库地址> --skill find-skills -g -y
```

本仓库不再维护两份 `find-skill` 副本：根目录 `skills/find-skill/` 是唯一源码，`.claude/skills/find-skill/` 只是通过符号链接暴露出来的 project-local 入口。

## Skills CLI 搜索示例

```bash
npx skills find github
npx skills find "web scraping"
npx skills find "react performance"
```

## 如何评估搜索结果

| 维度 | 看什么 | 差 | 好 |
|------|--------|-----|-----|
| 相关性 | 描述和你的需求是否匹配 | 勉强相关 | 精准命中 |
| 活跃度 | 最近更新时间 | 1年前 | 3个月内 |
| 质量 | README 是否清晰，代码是否规范 | 混乱 | 结构清晰 |
| 依赖 | 需要什么外部依赖 | 复杂 | 零依赖或轻量 |
| 许可证 | 能否商用 | 不明确 | MIT/Apache |

如果目标仓库的许可证不明确，先不要直接复制实现。先看：[什么是许可证？](../01-概念入门/02-什么是-许可证.md)

**找到现成方案后**：安装或 Fork → 试跑 → 评估是否满足需求 → 决定直接用还是改造。

**找不到现成方案**：恭喜，你是第一个。参考 03 基于模板创建 开始搭建。

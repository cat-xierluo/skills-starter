# 02 搜索现成方案

不要重复造轮子。动手之前先搜一圈。

## 为什么先搜索？

- 很可能已经有人解决了类似问题
- 站在前人肩膀上，省下的是你自己的时间
- 搜索过程中会加深对问题的理解
- 有时候会发现更优雅的解决思路

## 搜索顺序

**1. 独立 `find-skills` 项目或 Skills CLI（首选）**
先用现成生态做第一轮筛选。

**2. GitHub**
关键词 + `openclaw skill` 或 `claude skill` 搜索。重点看 stars 和最近更新时间。

**3. 互联网**
通用问题（API 用法、概念理解）用搜索引擎。

## `find-skills` 的两种用法

### 方式一：直接用 Skills CLI 搜索

```bash
npx skills find <关键词>
```

### 方式二：先安装独立 `find-skills` 项目

如果你的工作流依赖独立 Skill 版本，先按你选用仓库的 README 安装：

```bash
npx skills add <仓库地址> --skill find-skills -g -y
```

本仓库的 `skills/find-skill/SKILL.md` 负责说明搜索流程和筛选标准，不等同于外部项目源码。

## Skills CLI 搜索示例

```bash
npx skills find github
npx skills find "web scraping"
npx skills find "react performance"
```

## 如何评估搜索结果

| 维度 | 看什么 | 差 | 好 |
|------|--------|-----|-----|
| 相关性 | 描述和你的需求是否匹配 | 勉强相关 | 精准命中 |
| 活跃度 | 最近更新时间 | 1年前 | 3个月内 |
| 质量 | README 是否清晰，代码是否规范 | 混乱 | 结构清晰 |
| 依赖 | 需要什么外部依赖 | 复杂 | 零依赖或轻量 |
| 许可证 | 能否商用 | 不明确 | MIT/Apache |

**找到现成方案后**：安装或 Fork → 试跑 → 评估是否满足需求 → 决定直接用还是改造。

**找不到现成方案**：恭喜，你是第一个。参考 03 基于模板创建 开始搭建。
